# 非機能仕様書

## 1. 技術スタック
- Next.js
- TypeScript
- Tailwind CSS
- Supabase
- PostgreSQL
- Vercel

必要に応じて:
- shadcn/ui
- Zod
- React Hook Form

## 2. アーキテクチャ方針
- フロントエンド: Next.js App Router
- バックエンド: Next.js Route Handlers
- DB: Supabase Postgres
- OGP取得: サーバー側API

## 3. OGP取得に関する制約
- クライアント側から直接対象URLを取得しない
- CORS回避のため、サーバー側で fetch / parse を行う
- robotsや規約に反する収集を目的としない
- ユーザーが明示的に入力したURLのみ対象にする

## 4. パフォーマンス
- 記事一覧100件程度で快適に表示できること
- URL追加時のメタ情報取得は体感上数秒以内が望ましい
- 画像読み込みは遅延読込を考慮する

## 5. セキュリティ
- ユーザーごとに記事データを分離する
- DBアクセスは認証前提で制御する
- メタ情報取得APIは外部URLの扱いに注意し、SSRFリスクを考慮する

## 6. 可用性
- Vercel上でデプロイ可能であること
- 環境変数は Vercel Env で管理する

## 7. 保守性
- ドメインモデル（Article）を中心に設計する
- OGP取得ロジックと記事CRUDロジックを分離する
- UIコンポーネントとDBアクセス層を分離する

## 8. 拡張性
将来的に以下を追加できる構造にする

- プロパティのユーザー定義化
- レコメンド
- Chrome拡張
- ブックマークレット
- 自動分類
- SNS / times / Notion 連携

## 9. MVPでの割り切り
- 完璧なOGP取得は保証しない
- 取得失敗時もURLのみ保存可能とする
- クローリングや自動収集は対象外
- AI機能は対象外
