# 機能仕様書

## 1. 記事追加機能

### 1.1 URL入力
- ユーザーはURL入力欄に記事URLを入力できる
- 貼り付け操作を主導線とする

### 1.2 メタ情報取得
URL入力後、同一画面内で対象URLのメタ情報取得を実行する

取得対象:
- title
- description
- imageUrl
- siteName

取得方法:
- サーバー側API経由でHTMLを取得し、OGPやtitleタグを解析する

### 1.3 インラインプレビュー
取得成功時、画面遷移せず、その場にプレビューカードを表示する

表示内容:
- 画像
- タイトル
- 軽い説明
- ドメインまたはサイト名

### 1.4 保存
保存ボタン押下で Article を登録する

デフォルト値:
- status = inbox
- note = 空
- tags = なし

### 1.5 取得失敗時
メタ情報取得に失敗しても、以下の扱いとする

- エラーメッセージを表示
- URLのみで保存可能
- title は空またはURLを暫定表示

---

## 2. 記事一覧機能

### 2.1 一覧表示
登録済み記事をカード一覧で表示する

カード表示項目:
- imageUrl
- title
- description
- status
- tags
- siteName
- updatedAt

### 2.2 表示形式
MVPではカード型一覧を採用する
ブクログやKindleのように画像ベースで俯瞰できることを重視する

### 2.3 並び順
初期状態では新しい順で表示する

将来的な並び替え候補:
- 更新日順
- 作成日順
- 状態順

---

## 3. 記事詳細機能

### 3.1 詳細表示
記事カード選択時に詳細領域または詳細画面を表示する

表示項目:
- タイトル
- URL
- description
- image
- siteName
- status
- note
- tags

### 3.2 メモ編集
- note を自由入力で編集できる

### 3.3 状態変更
- status を切り替えできる

### 3.4 タグ編集
- 複数タグを付与・削除できる

---

## 4. タグ機能

### 4.1 タグ追加
ユーザーは任意のテキストタグを追加できる

### 4.2 タグ削除
ユーザーは記事からタグを外せる

### 4.3 タグ一覧
MVPではグローバルタグ管理画面は必須としない
ただし、一覧絞り込みのためにタグ単位の抽出は可能にする

---

## 5. 状態フィルタ機能
記事一覧に対して status ベースで絞り込みできる

対象:
- all
- inbox
- reading
- thinking
- done

---

## 6. 検索 / フィルタ
MVPでは以下を対象とする

- キーワード検索（title / note）
- status フィルタ
- tag フィルタ

---

## 7. 記事リンク遷移
詳細画面またはカードから元記事を開ける

---

## 8. 削除機能
ユーザーは不要な記事を削除できる

---

## 9. 認証機能
MVPではユーザーごとの記事管理を前提とするため、認証を持つ

候補:
- Supabase Auth
- メールログインまたは簡易ログイン

ただし、実装優先順位によっては認証を後段に回し、
先に単一ユーザー前提の土台を作ってもよい
