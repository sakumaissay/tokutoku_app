# 要求事項書

## 1. 背景
ユーザーは日常的に X（Twitter）、Google Discover、Instagram、検索結果などからWeb記事に触れている。
記事を見つけた際、その場ですぐに感想やコメントを言語化できる場合は Slack の times に投稿できるが、以下のような場合は整理が後回しになる。

- 内容が重く、その場で読み切れない
- なぜ気になったかを即座に言語化できない
- 仕事関連・私用などで保管先を分けたい
- 後で読み返してから判断したい

現状は自分のDMなどに URL を置く運用で代替しているが、以下の課題がある。

- 一覧性が低い
- 視覚的に見返しづらい
- 状態管理（未読、読みかけ、考え中など）ができない
- 情報を“とりあえず入れる”導線は軽いが、後から整理しにくい
- Notion でも代替可能だが、入力負荷と設計負荷が高い

## 2. 解決したい課題
本プロダクトは、Web記事を「保存する」だけでなく、あとで整理・思考・活用しやすい状態でストックすることを目的とする。

特に重視するのは以下である。

1. とりあえず突っ込めること
2. あとから整理できること
3. 視覚的に見返しやすいこと
4. 思考やコメントの前段として使えること

## 3. プロダクト定義
tokutoku は、気になったWeb記事を URL ベースで素早く取り込み、
タイトル・説明・画像などのメタ情報を自動取得しながら、
後から整理・分類・メモできる記事ストックアプリである。

## 4. コンセプト
- 「あとで読む」より「あとで活かす」
- 保存庫ではなく、中間ストック
- 完成した知識ではなく、未整理の気づきをためる場所
- DM と Notion の中間にある、軽くて整理しやすい記事置き場

## 5. コア価値
### 5.1 入力コストが低い
URL を貼るだけで記事候補として取り込める

### 5.2 視覚的に把握しやすい
タイトル・説明・画像付きカードで一覧表示できる

### 5.3 あとから整理しやすい
状態、タグ、メモなどを使って後から整理できる

### 5.4 発信・活用の前段になる
times 投稿や社内ブログ化の前に、一時的に貯めて熟成できる

## 6. MVPのスコープ
MVPでは以下に集中する。

- URLベースの記事追加
- OGP等のメタ情報取得
- 記事カード一覧表示
- 状態管理
- タグ・メモ付与
- フィルタ・一覧整理

以下はMVP対象外とする。

- Web全体からの自動収集
- クローリング
- レコメンド
- ブックマークレットや拡張機能
- SNS連携
- AI要約やAI分類
